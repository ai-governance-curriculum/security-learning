# Role Requirements: AI Infrastructure Security Engineer

**Level:** 2.5
**Experience Level:** 10+ years in cybersecurity with 2-3 years specifically in AI/ML security or willingness to specialize. CASP+ certification valuable.
**Market Demand:** Critical - Severe shortage of professionals combining deep security expertise with AI/ML knowledge. Increasing regulatory scrutiny (AI Act, etc.) driving explosive demand. Essential role as AI systems handle sensitive data and critical decisions. High demand from healthcare, financial services, government, and tech companies.
**Salary Range (US, 2025):** $140,000 - $280,000 (median $190,000)

## Core Responsibilities
- Design and implement security architecture for AI/ML systems
- Protect ML models from adversarial attacks and data poisoning
- Implement compliance frameworks for AI systems (GDPR, HIPAA, SOC2)
- Conduct security assessments and penetration testing of ML infrastructure
- Design and enforce data privacy controls for training and inference data
- Implement model security including input validation and output filtering
- Build audit logging and compliance monitoring systems for AI operations
- Manage secrets, credentials, and access controls for ML infrastructure
- Implement network security for ML systems (zero-trust architecture)
- Design encryption strategies for data at rest and in transit
- Conduct security reviews of ML pipeline and deployment processes
- Develop incident response procedures for AI security events
- Monitor for model theft and intellectual property protection
- Ensure supply chain security for ML dependencies and models
- Implement responsible AI practices and bias detection

## Technical Skills
### Required
- Deep cybersecurity expertise (10+ years preferred)
- Knowledge of ML systems and their unique security challenges
- Security architecture and design patterns
- Compliance frameworks (GDPR, HIPAA, SOC2, ISO 27001)
- Network security and zero-trust architecture
- Identity and access management (IAM, RBAC, ABAC)
- Encryption technologies and key management
- Security monitoring and SIEM tools
- Penetration testing and vulnerability assessment
- Cloud security (AWS, GCP, Azure)

### Preferred
- Knowledge of OWASP ML Security Top 10
- Adversarial ML and model robustness techniques
- Differential privacy and federated learning security
- Model explainability and interpretability tools
- Data governance and privacy-preserving ML
- Secure multi-party computation (SMPC)
- Homomorphic encryption awareness
- AI red teaming experience
- Regulatory technology (RegTech) platforms
- Experience with CASP+ or similar advanced certifications

## Soft Skills
- Risk assessment and threat modeling
- Communication of security risks to non-technical stakeholders
- Balancing security with usability and performance
- Collaboration with ML teams and business units
- Staying current with emerging threats
- Incident management and crisis communication
- Policy development and enforcement
- Training and security awareness
- Attention to detail
- Strategic thinking for long-term security posture

## Prerequisites
- Bachelor's degree in Computer Science, Cybersecurity, or related field
- Strong background in enterprise security
- Understanding of machine learning concepts and lifecycle
- Experience with cloud security and infrastructure
- Knowledge of compliance and regulatory requirements
- Security certifications (CISSP, CISM, or equivalent)

## Day-to-Day Activities
- Conducting security assessments of ML systems and pipelines
- Implementing and monitoring security controls
- Reviewing ML code and infrastructure for security vulnerabilities
- Responding to security incidents involving AI systems
- Designing compliance frameworks and audit procedures
- Testing ML models for adversarial robustness
- Managing access controls and secrets for ML infrastructure
- Creating security documentation and runbooks
- Training teams on AI security best practices
- Evaluating third-party ML tools for security risks
- Monitoring security metrics and generating compliance reports

## Success Metrics
- Zero security breaches or data leaks in ML systems
- Compliance audit pass rates (100% target)
- Time to detect and respond to security incidents
- Coverage of security controls across ML infrastructure
- Vulnerability remediation time
- Security training completion rates for ML teams
- Number of security issues identified in reviews
- Model robustness against adversarial attacks
- Successful penetration test results

## Common Challenges
- Balancing security requirements with ML team velocity
- Addressing unique security challenges of ML systems (model theft, poisoning)
- Keeping up with rapidly evolving AI threat landscape
- Implementing privacy-preserving ML techniques
- Managing compliance across multiple jurisdictions
- Securing complex ML supply chains
- Detecting subtle adversarial attacks on models
- Building security awareness in data science teams
- Navigating ambiguous regulatory requirements for AI

## Key Technologies
- **Programming Languages:** Python (for ML security tooling), Go or Rust (for security tools), SQL (for audit queries), Bash/PowerShell (for security automation)
- **ML Frameworks:** Understanding of PyTorch, TensorFlow (for security analysis), Knowledge of model architectures and vulnerabilities, Adversarial ML frameworks (CleverHans, Foolbox, ART)
- **Infrastructure:** Kubernetes security (Pod Security Policies, Network Policies), Cloud security services (AWS Security Hub, Azure Security Center), Container security (Aqua, Twistlock, Sysdig), Zero-trust network architecture
- **DevOps Tools:** Security-focused CI/CD (SAST, DAST, dependency scanning), GitOps with security controls, Infrastructure scanning (Terraform security, CloudFormation Guard), Policy as Code (OPA, Sentinel)
- **MLOps Tools:** Secure ML platforms with audit capabilities, Model governance platforms, Data lineage and provenance tracking, Secure model registries
- **Monitoring & Observability:** SIEM platforms (Splunk, QRadar, LogRhythm), Security monitoring (Datadog Security, Sumo Logic), Model monitoring for anomalous behavior, Audit logging and compliance reporting, Threat detection platforms
- **Data Processing:** Data masking and anonymization tools, Privacy-preserving analytics, Secure data pipelines, Data loss prevention (DLP) tools
- **GPU Tools:** Awareness of GPU security considerations, Secure GPU resource allocation
- **Other Tools:** Secrets management (HashiCorp Vault, AWS Secrets Manager), IAM and access control (Okta, Auth0, cloud IAM), Encryption tools and KMS, Vulnerability scanners (Nessus, Qualys, Snyk), Penetration testing tools (Metasploit, Burp Suite), Compliance platforms (Vanta, Drata, Secureframe), Model watermarking and fingerprinting tools
